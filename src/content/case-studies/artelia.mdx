The hard part was not drawing ships on a map. It was turning difficult AIS data into something a person could explore and trust. In 2018, I designed and developed a maritime traffic tool for Artelia and Total that analysed activity in Abidjan’s port.

I was responsible for the user experience and the build. Over three months, I turned raw maritime signals into a database and a web application, using D3.js for the visualisations and Mapbox and Leaflet for the map.

<ProjectImage src={mapImage} alt="Maritime traffic map and analytical interface" caption="The map brings vessel movements and the surrounding analytical interface into one view." />

The data source needed as much care as the interface. I used web scraping to gather information on roughly 1,300 ships from three sources, using BeautifulSoup to build a dataset that could be mapped correctly.

<ProjectImage src={databaseImage} alt="Ship database interface created for the maritime traffic tool" caption="The ship database connects the source data to the decisions a user can make in the interface." />

The visualisation challenge was making the data useful, not merely decorative. This project taught me a great deal about geodata, maritime traffic, and how an interface can surface insight from a complex system. Some details remain confidential, but these screens show the work that was possible to share.
